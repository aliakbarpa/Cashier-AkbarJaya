Common Issues & Solutions

‚ùå "Python is not recognized"

Python is not installed or not in PATH

Solution:

  • Download Python from python.org
  • During installation, check "Add Python to PATH"
  • Restart Command Prompt after install

‚ö†Ô∏è "Windows protected your PC"

Windows SmartScreen warning (first run)

Solution:

  • Click "More info"
  • Click "Run anyway"
  • This is normal for new EXE files
  • Won't appear on subsequent runs

‚ö†Ô∏è Antivirus flags the EXE

False positive (very common with PyInstaller)

Solution:

  • Add exception in antivirus software
  • Whitelist the file or folder
  • This is NOT a virus - it's a known PyInstaller issue

‚ùå Build fails with errors

Previous build files may be corrupted

Solution:

# Delete old build files
rmdir /s /q build
rmdir /s /q dist
# Try building again
python build_exe_script.py

‚ùå EXE won't run on another PC

Missing data folders or wrong architecture

Solution:

  • Ensure data/ folder exists next to EXE
  • Check that CSV files are in data/
  • Try running as Administrator
  • Build on same architecture (32 vs 64-bit)

üí° Pro Tip: Testing

Always test your EXE on a computer without Python installed before distributing. This ensures it truly works standalone!
